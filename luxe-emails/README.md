# luxe-emails

Branded transactional and internal email templates for Student Luxe and Stay Luxe.

This folder is self-contained and ready to be lifted into its own `luxe-emails`
repository. It currently holds the redesigned **Lead Qualified** internal
notification that senior staff receive when a lead is moved to `Qualified` on
the Monday Leads board (`2171015719`).

## What's here

```
luxe-emails/
  templates/
    lead-qualified.html            # static mockup with sample data (open in a browser)
    lead-qualified.generated.html  # output of the render function (regenerated by preview.js)
    preview.png                    # rendered screenshot of the mockup
  src/
    render-lead-qualified.js       # pure render function: lead object -> { subject, html }
    sample-data.js                 # example payload
    preview.js                     # node src/preview.js -> writes the generated HTML
```

## Preview

```bash
node src/preview.js
# then open templates/lead-qualified.generated.html in a browser
```

Or just open `templates/lead-qualified.html` directly for the hand-built mockup.

## The redesign

The old automation email was a flat list of fields. The redesign turns it into
a **lead intelligence card** that lets a senior staffer judge the lead in five
seconds, matched to the existing Student Luxe brand (navy `#0d1a2e`, gold
`#B8966E`, cream `#f4f1ec`, DM Sans + Georgia serif, 600px rounded card).

### Metrics shown

Requested:

- **Lead cooking time** - time from lead created to qualified, with a
  comparison against the team average, plus a created-to-qualified mini
  timeline in the hero.
- **Lead source** - channel plus the campaign / adgroup it came from.
- **Person assigned** - the lead owner, shown distinctly from who qualified it.
- **Sales progress notes** - the full thread of Monday updates as a timeline.

Added (answers to "what else could it have"):

- **Qualified by vs assigned to** - accountability on both sides.
- **Stay details** - check-in, check-out, length of stay, guests, location,
  budget (with flex), contact number.
- **Visited paths** - the `visited paths` column from the Leads board, rendered
  as the page journey that led to the enquiry.
- **Next action + due date** - what happens next so the lead does not stall.
- **One-tap CTAs** - Open in Monday, Call guest, WhatsApp guest.
- **Smart subject line** - guest, cooking time, and owner up front.

### Deliberately left out

These are not reliably known at the moment the lead is qualified, so they were
dropped to keep every figure honest:

- Estimated booking value (not entered until later in the pipeline).
- Lead score / temperature.
- Speed to first contact.

### Further ideas (not yet built)

- Cold-lead SLA badge if cooking time crosses a threshold.
- Est. booking value once it is captured later in the pipeline.
- "Similar booked leads" for pricing confidence.
- Brand switch (Stay Luxe palette) driven by a `brand` field, to keep parity
  with the two submit handlers in `luxe-lead-capture`.

## Wiring it up

`renderLeadQualified(lead)` is pure - no network, no secrets. Feed it a
normalised lead object (see the JSDoc in `src/render-lead-qualified.js`) and
POST the returned `html` + `subject` to Resend, exactly the way
`api/weekly-summary.js` does in `luxe-lead-capture`:

```js
const { renderLeadQualified } = require('./src/render-lead-qualified');
const { subject, html } = renderLeadQualified(lead);

await fetch('https://api.resend.com/emails', {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${process.env.RESEND_API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    from: 'Student Luxe <reservations@studentluxe.co.uk>',
    to: ['senior-staff@studentluxe.co.uk'],
    subject,
    html
  })
});
```

Trigger source: a Monday automation / webhook that fires when an item's status
changes to `Qualified`. Map the Monday columns (`color_mkxk8y67` source,
`people98` salesperson, `created_at`, status, and the item updates feed) onto
the `lead` object fields before calling the renderer.

## Conventions

- Complete files always.
- No em dashes anywhere.
- Table-based, inline-styled HTML for email-client compatibility.
- Keep Student Luxe and Stay Luxe in visual parity.
